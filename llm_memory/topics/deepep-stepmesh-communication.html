摘要

梳理 EP dispatch/combine、All-to-All、NVSHMEM/RDMA、StepMesh M2N、A/F 分离和 collective 基础。

中文正文

  • DeepEP 和 StepMesh 代表 MoE/serving 中通信成为一等公民：dispatch/combine、All-to-All、RDMA 和 topology 都会进入性能模型。
  • EP 通信要区分 normal/high-throughput 与 low-latency kernel，不同模式分别匹配 prefill 和 decode 的吞吐/延迟目标。
  • StepMesh/M2N、TensorDirect、A/F 分离等方案都在尝试减少跨角色、跨机或跨层的数据搬运成本。
  • 通信优化不只是带宽问题，还包括 buffer 管理、overlap、rank mapping、拥塞控制、错误恢复和可观测性。
  • 设计 serving 拓扑时，应把模型并行、expert 并行、PD 分离和物理网络拓扑同时画出来，避免局部优化导致全局瓶颈。
