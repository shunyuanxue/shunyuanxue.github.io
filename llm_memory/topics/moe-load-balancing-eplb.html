摘要

说明专家并行中的负载不均、冗余专家、在线 EPLB、runtime migration 和 serving-path 里的专家放置问题。

中文正文

  • MoE serving 的瓶颈常来自 expert 负载不均，而不是理论 FLOPS；热门 expert 会造成 dispatch/combine 排队和尾延迟。
  • EPLB 通过在线统计、冗余 expert、重映射或迁移，把 expert placement 放进 serving control loop。
  • 负载均衡要同时考虑 token routing 分布、expert compute、通信带宽、rank failure、batch shape 和缓存局部性。
  • 冗余 expert 可以降低热点，但会增加显存和一致性成本；迁移可以提升均衡度，但会带来拓扑重建和短期抖动。
  • MoE 调优应和 DeepEP、DP Attention、PD layout、autoscaling 联合评估，单独看 expert 平均负载容易误判。
