vLLM Scheduler

摘要

解释 vLLM V1 scheduler 的 waiting/running queues、resource budget、priority 和 prefill/decode 约束。

中文正文

  • vLLM scheduler 决定每个 engine step 计算哪些请求和哪些 token，它是吞吐、延迟和显存压力之间的协调器。
  • 核心队列通常包括 waiting 与 running；调度输出受 token budget、KV block budget、preemption、chunked prefill 和 speculative state 约束。
  • V1 scheduler 的重要变化在于更系统地连接 request state、cache block 和 execution output，适合和 PD/KV 页面一起阅读。
  • 优先级调度不能只做队列排序，还要定义抢占、starvation 保护、SLO class、tenant quota 和 recompute 成本。
  • 在 PD serving 中，prefill scheduler 和 decode scheduler 的目标不同，必须通过 router 和 KV manager 交换容量与状态。
