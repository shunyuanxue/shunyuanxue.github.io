摘要

描述 silent data corruption 的检测、隔离、诊断、硬件 quarantine 和业务侧质量 validator。

中文正文

  • SDC 是不立即 crash 却悄悄污染数值结果的硬件/低层计算错误，在训练和推理中都可能造成质量异常。
  • AI 系统里的 SDC 可能表现为 NaN、梯度爆炸、参数污染、黑帧、坏生成或难以复现的质量回退。
  • 治理路径包括检测、定位、隔离、降级、quarantine、case library 和业务侧 quality validator。
  • 低误报和低开销很关键；如果检测成本过高，就无法覆盖线上高吞吐路径。
  • SDC 应进入稳定性平台，而不是只放在硬件团队手册里，因为最终影响会传导到模型输出和用户体验。
