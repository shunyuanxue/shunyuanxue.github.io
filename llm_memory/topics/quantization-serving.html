摘要

梳理 FP8、W4A8、KV-cache 量化、DS-R1 W4A8KV8 workflow、scale 布局和线上质量校验风险。

中文正文

  • 推理量化的目标是在可接受质量损失下减少显存、带宽和算子成本，常见路径包括 FP8、INT8、INT4、W4A8 和 KV-cache quantization。
  • 权重量化、激活量化和 KV 量化的风险不同：权重影响模型能力，激活影响动态范围，KV 影响长上下文和多轮会话稳定性。
  • DS-R1 W4A8KV8 这类 workflow 要同时验证 scale 布局、算子支持、calibration 数据、online quality guard 和 fallback 策略。
  • MoE 量化还要考虑 grouped GEMM、expert routing、热专家分布和通信 payload，不能只看单层 GEMM benchmark。
  • 线上推广应采用灰度、质量回放、异常样本分析和按模型/任务分层的阈值，而不是只依赖平均 perplexity 或离线吞吐。
