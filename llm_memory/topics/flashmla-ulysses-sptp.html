摘要

关注 MLA decode kernel、Hopper 调度、KV paging、Ulysses 序列并行和 MLA prefill 负载均衡。

中文正文

  • FlashMLA 关注 MLA decode kernel 的实际效率，核心在 KV paging、SM load balance、Hopper WGMMA/TMA 和 memory scheduling。
  • Ulysses/SPTP 代表序列并行和张量并行在长上下文/MLA 场景下的组合方式，目标是平衡计算与通信。
  • MLA kernel 不是孤立优化：o_proj sharding、KV page block、scheduler budget 和 prefill/decode 切分都会影响端到端收益。
  • Hopper 特性只有在 tile、pipeline、TMA 和 occupancy 设计正确时才会释放，否则可能被 memory movement 吃掉。
  • 这类页面应作为 kernel/parallelism 层参考，向上连接 DeepSeek 性能模型，向下连接硬件网络页面。
