This repository is an AI infrastructure memory archive. Treat the existing .docx, .pdf, .png, and .drawio files as source material, and treat wiki/ as the maintained knowledge layer.

Working Contract

  • Start from wiki/index.md before answering broad questions.
  • Prefer updating existing wiki pages over creating near-duplicate pages.
  • Do not rewrite or move original source documents unless the user explicitly asks for an archival migration.
  • When adding claims, cite source paths from wiki/sources/catalog.md or the original file path.
  • If a source appears to be image-heavy or extraction quality is low, mark that in the page instead of pretending the text was fully parsed.
  • Keep pages concise and link out to source documents for detail.
  • Update wiki/log.md whenever a meaningful ingest, reorganization, or conclusion update is made.

Page Schema

Each maintained wiki page should start with YAML front matter:

---
title: Short title
status: draft | active | deprecated
updated: YYYY-MM-DD
tags: [tag-a, tag-b]
sources:
  - relative/path/to/source.docx
---

Use the following section order when it fits:

  1. Summary
  2. Key Ideas
  3. Architecture
  4. Tradeoffs
  5. Open Questions
  6. Source Notes

Ingest Rules

  • Keep raw extraction separate from synthesis. Source files explain what was said; wiki pages explain what it means and how it connects.
  • If multiple source files overlap, synthesize the newest or most complete version and list older versions in Source Notes.
  • If two sources conflict, keep both claims and mark the disagreement.
  • Link repeated concepts back to canonical topic pages, such as wiki/topics/pd-disaggregation.md or wiki/topics/agentic-rl-infra.md.
