这是一个面向 AI Infra 的中文技术 blog / 本地知识库。仓库保留原始 source documents，同时在 wiki/ 下维护一层可持续更新的中文主叙述知识层。

网页版本采用 GitHub Pages 友好的 blog 形式，以及“中文解释 + 英文术语保留”的方式：中文负责阅读路径和结论， LLM Serving、PD Disaggregation、KV Cache、Agentic RL 等英文术语保留， 方便继续对照源码、论文、PR 和内部文档。

双版本查看方式

公开版只包含中文综合 wiki、source catalog 和脱敏后的 source index，适合发布到 GitHub Pages：

npm install
npm run build:sources
npm run build:site

然后在浏览器中打开 site/index.html，或启动本地预览：

npm run serve:site

本地预览地址默认是 http://127.0.0.1:4173/index.html。生成结果是纯静态 HTML、CSS 和 JavaScript，首页是 blog 索引，文章页是技术 blog 阅读版式。

私有完整版包含 wiki/sources/extracted/ 的原文转写、章节索引、代表性片段和 全文搜索，只建议在本地或 private repo 中查看：

npm run build:private-site
npm run serve:private-site

私有预览地址默认是 http://127.0.0.1:4174/index.html。维护时推荐用 VSCode 编辑 markdown，用 private_site/ 做阅读和检索界面，用 site/ 做公开发布产物。

如果需要把 wiki/ 源文档统一刷新成中文主叙述结构，可以运行：

npm run rewrite:wiki

tools/localize_wiki_sources.mjs 是早期只添加中文导读的兼容脚本；当前主线使用 tools/rewrite_wiki_chinese_main.mjs，会把可见正文改成中文主体。

如果新增、删除或重命名了原始 source documents，可以刷新 source 层索引：

npm run build:sources

其中 wiki/sources/catalog.md 是平铺目录，wiki/sources/classified-index.md 按照 wiki 主题建立分层阅读入口，wiki/sources/extracted/ 是每个原始文档的 私有 markdown 转写页。所有这些步骤都不移动原始文件。

如果使用 Codex 内置 runtime，而不是本地 npm install，可以运行：

NODE_PATH=/Users/bytedance/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/node_modules \

  tools/build_static_site.mjs

发布到 GitHub Pages

  1. 创建或使用现有 GitHub 仓库。
  2. 将本仓库 push 到 origin。
  3. 在 GitHub Settings -> Pages 中选择 GitHub Actions 作为发布来源。
  4. .github/workflows/pages.yml 会在 push 到 main 或 master 时构建 site/ 并部署。构建产物包含 .nojekyll，因此 GitHub Pages 会按普通 静态站点发布，而不是交给 Jekyll 再处理。

当前重点

当前知识库以 AI Infra 为中心，重点包括：

  • LLM 推理服务、PD 分离、KV Cache、调度器和框架设计。
  • Agentic RL 基础设施、rollout 系统、sandbox execution、Fornax 和 Swalm。
  • 多角色推理系统的 Kubernetes-style Operator / CRD 编排。
  • 稳定性、可观测性、故障处理和集群级可靠性。
  • 面向训练和推理的 GPU/NPU/network/communication 基础设施。

原始 source files 仍保留在仓库根目录和 lark_docs/。wiki/ 是后续综合、 检索和维护的主层。
