摘要

记录重复源的优先级、低抽取/图片型源状态、PDF 文本抽取方法和剩余视觉复核工作。

中文正文

  • 源文件治理负责决定哪些原始文件可信、哪些是重复、哪些抽取质量低、哪些需要 OCR 或人工复核。
  • 原始文件的主题归档入口在 原始文档分层索引，那里按 wiki 主题列出原文档、关联中文页面和复核状态。
  • 重复文件要记录优先级：新版本、完整版本、权威版本和历史版本的角色要分清。
  • 低抽取文件不能直接作为强结论来源，应先标注风险，再通过截图/OCR/人工读图补证据。
  • 治理页面应持续更新，避免 wiki 结论和 source material 之间失去可追溯关系。

阅读和维护建议

  • 先把本页当作中文结论层阅读；需要实现细节、原始数字或图示时，再回到下方 source documents。
  • 英文术语保留是为了方便继续搜索源码、论文、PR、issue 和内部文档，不表示正文仍以英文为主。
  • 如果后续新增资料与本页结论冲突，应保留两种说法并在来源说明里标记差异。
