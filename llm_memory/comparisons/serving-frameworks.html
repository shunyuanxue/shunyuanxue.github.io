摘要

横向比较 vLLM、SGLang、xLLM、AIbrix、Dynamo、DistAgg、Inferobust、veFuser 的调度、KV、PD 和控制面。

中文正文

  • 框架对比应按能力维度比较，而不是按品牌：scheduler、KV/cache、PD、MoE、communication、control plane、observability 和 autoscaling。
  • vLLM 强在通用推理 runtime、PagedAttention 和生态；SGLang 强在 structured serving、PD/DP Attention/EPLB 等快速演进能力。
  • xLLM、Dynamo、AIbrix、DistAgg、Inferobust、veFuser 分别代表 runtime overhead、PD/control plane、KV transfer、fault handling 和 DAG serving 的不同侧面。
  • 没有单一框架覆盖所有场景，真实平台通常需要按 workload 组合 runtime、router、operator 和观测系统。
  • 做技术选型时应先明确模型形态、SLO、硬件、租户隔离、故障策略和团队维护能力，再映射到框架能力。
