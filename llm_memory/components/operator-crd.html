摘要

推理多角色、多 shard、多 Worker 的 Kubernetes-style 控制面抽象和生命周期管理。

中文正文

  • Operator/CRD 把多角色推理服务声明化，使 Application、WorkerSet、Role、Worker 等资源可以被控制器持续调谐。
  • 多角色 serving 不能只靠 Deployment/Service，需要表达 shard、replica mode、pooled mode、role dependency 和 stateful lifecycle。
  • 控制面要把 desired state、observed state、Conditions、revision、autoscaling 和 rollout strategy 统一起来。
  • CRD 设计应避免把所有字段塞进一个大 spec，而要按角色、网络、资源、状态和策略分层。
  • Operator 的价值在于把复杂运维动作变成可审计、可重试、可回滚的状态机。
