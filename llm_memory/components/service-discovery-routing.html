摘要

覆盖 Worker 内部 DNS、headless service、endpoint 到 service 映射、多租户隔离和动态路由。

中文正文

  • 服务发现与路由负责把抽象角色映射到可调用 endpoint，使 router、worker、prefill pool、decode pool 和 control plane 能互相定位。
  • Kubernetes 内部常用 headless service、DNS、label selector 和 endpoint mapping，但多角色 serving 还需要理解业务拓扑。
  • 路由策略要感知 tenant、model、role、capacity、KV location、cache affinity 和 failure state。
  • 动态扩缩容时，endpoint 变化必须和 readiness、drain、session stickiness、backpressure 协调，否则容易产生黑洞流量。
  • 路由层是 SLO 和资源效率的交汇点，不能只做四层负载均衡。
