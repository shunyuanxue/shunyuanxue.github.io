摘要

抽取 Application、Worker、Role、WorkerSet、status、autoscaling 字段，作为 schema 查询页。

中文正文

  • CRD schema 是多角色 serving 的契约文档，说明 Application、Worker、Role、WorkerSet 等对象如何表达业务拓扑。
  • spec 侧描述期望：模型、镜像、资源、role、replica、network、storage、autoscaling 和更新策略。
  • status 侧描述现实：ready、available、progressing、degraded、terminating、reason、message 和 observedGeneration。
  • schema 应尽量稳定，把扩展点留给 policy、template 和 Conditions，避免频繁破坏兼容性。
  • 阅读本页时可把它当作实现 Operator、debug 控制面和制定平台 API 的字段索引。
