复盘 28K-128K 长文总结分图服务不可用事故，强调 RPC timeout、KV cleanup 和控制面容错。

中文正文

  • 长上下文分图延迟事故暴露的问题不是单个 timeout，而是分图服务、RPC、KV cleanup、重试语义和控制面状态不一致的组合。
  • 28K-128K 长文总结这类请求会放大 prefill、KV、网络和 scheduler 的尾部风险。
  • 如果失败请求已经影响 KV/cache，简单重试可能制造重复计算、脏状态或资源泄漏。
  • 事故复盘应记录请求输入长度、分图阶段、RPC timeout、cleanup 结果、worker 状态和用户可见错误。
  • 修复方向包括更细的超时分层、幂等协议、失败清理、drain/restart 策略和端到端 trace。
