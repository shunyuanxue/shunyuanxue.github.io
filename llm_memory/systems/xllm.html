摘要

xLLM 的 zero-overhead runtime、分图推理、KV Cache Store 和 VLM encoder 分离设计。

中文正文

  • xLLM 关注 zero-overhead runtime、分图推理、KV Cache Store 和 VLM encoder 分离。
  • zero-overhead 的目标是减少框架层调度/复制/同步开销，让模型执行路径更贴近硬件能力。
  • KVCacheStore 把 cache 从单进程状态提升为可管理资源，使分图、PD、session 迁移和 prefix reuse 更容易组合。
  • VLM encoder 分离说明多模态 serving 应把 encoder、decoder、cache 和 router 的边界明确画出来。
  • xLLM 可以和 vLLM/SGLang 对比：它强调 runtime overhead 和系统边界，而不仅是 kernel 或 scheduler。
